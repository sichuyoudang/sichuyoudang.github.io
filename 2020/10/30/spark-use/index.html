Spark 入门使用经验总结

Posted on 2020-10-30

环境： spark 2.4.0

SparkSession 和 SparkContext

每个 Spark 应用程序都需要一个 Spark 环境，这是 Spark RDD API 的主要入口点，在 Spark 的早期版本中，SparkContext 是 Spark 的主要切入点，因为 RDD 是主要的 API，我们通过 Sparkcontext 来创建和操作 RDD。

对于每个其他的 API，我们需要使用不同的 context。例如，对于 Streming，我们需要使用StreamingContext；对于 sql，使用 sqlContext；对于 Hive，使用 hiveContext。SparkSession 本质上是对这些环境的组合。

SparkSession 现在是读取数据，处理元数据，配置会话和管理集群资源的入口。

执行任务方式，spark-submit 和 spark-shell

Spark Shell 提供了一个名为 sc 的预配置 Spark 环境(SparkContext)和一个名为 spark 的预配置 Spark 会话(SparkSession)。 如果启动 spark shell 不写任何参数，则默认启动 local 模式。

rdd 和 dataframe

DataFrame 出现在Spark版本1.3中。可以将 DataFrame 称为数据集，将其组织为命名列。DataFrame 类似于 R/Python 中的关系数据库或数据框架中的表。它可以说是一个具有良好优化技术的关系表。DataFrame背后的想法是允许处理大量结构化数据。DataFrame 包含具有 Schema 模型的行。
DataFrame 优于 RDD，但也包含 RDD 的功能(不包含 map 功能)。RDD 和 DataFrame 的共同特性是不可变性，在内存中，弹性，分布式计算能力。它允许用户将结构强加到分布式数据集合上，从而提供更高级别的抽象。

引入 dataframe 之前，python 的查询速度普遍比使用 rdd 的 scala 查询慢，通常情况下，这种查询性能的降低是因为 Python 和 JVM 的通讯开销。dataframe 使用了 Catalyst 优化器（以及 Tungsten 项目），python 性能会有明显的改进。
