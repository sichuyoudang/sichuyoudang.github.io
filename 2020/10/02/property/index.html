Python 增加实例属性额外处理的3种方法

Posted on 2020-10-02

对实例属性的获取和设定上，想要增加一些额外的处理过程（如类型检查或验证），有如下几种方法：

1. 使用 property 属性方法

如上，一共有三个互相关联的方法，它们必须有相同负名称。第一个方法是一个 getter 函数，并且将 first_name 定义为 property 属性。其他两个方法将可选的 setter 和 deleter 函数增加到了 property 属性上。需要重点强调的是，除非 first_name 已经通过 @property 的方式定义为了 property 属性，否则是不能定义 @first_name.setter 和 @first_name.deleter 装饰器的。

使用 property 需要注意事项:

如果 property 没有完成任何额外的处理任务，就不要实现这种方式，因为一是代码会变啰嗦，对其他人来说比较困惑，二是这样会让程序变慢很多。特别是如果稍后要对某个普通属性增加额外处理步骤时，可以在不修改已有代码的情况下将这个属性提升为一个 property。因为访问一个属性的语法不会变（访问普通属性和访问 property 属性的代码写法一样）。

此外不要编写那种定义了大量重复性 property 的代码，这样的代码膨胀，容易出错，这种情况下使用描述符更好。

2. 使用描述符类

所谓描述符就是以特殊方法 __get__() __set__() __delete__()的形式实现了三个核心的属性访问操作（对应 get， set， delete）的类。这些方法通过接受类实例作为输入来工作，之后底层实例字典会根据需要适当进行调整。

要使用一个描述符，把描述符的实例放置在类的定义中作为类变量来用:

每个描述符方法都会接受被操纵的实例作为输入，再执行所请求的操作，底层的实例字典会调整。描述符的 self.name 属性会保存字典的健，通过这些健可以找到存储在实例字典的实际数据。对于大多数 Python 类的属性，描述符都提供了底层的魔法，包括 @classmethod, @staticmethod, @property 甚至 __solts__。

如果只是想访问某个特定的类中的一种属性，并对此做定制化处理， property 属性方法会更简单。在需要大量重用代码的情况下，描述符更加有用。
