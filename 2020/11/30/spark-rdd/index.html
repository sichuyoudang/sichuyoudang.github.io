Spark RDD 基本操作

Posted on 2020-11-30

RDD 基础概念

RDD 弹性分布式数据集，数据集被划分成块，执行时分发到执行器执行，它是没有 schema 的数据结构，所以可以随意混合 dict, list, tuple。

变换(Transformations):懒执行,变换只是一些指令集并不会去马上执行,需要等到有Actions操作的时候才会真正的计算结果，比如: map, flatMap, groupByKey, reduceByKey
操作(Actions): 立即执行, count，take，collect，top，first

操作(Actions)

collect()方法的作用是收集分布在各个worker的数据到driver节点
take(n) 返回单个数据分区的前n 行
count() 返回 RDD 里面数据量

变换(Transformations)

map操作

flatMap 操作

map 是将一个长度为N的RDD转换为另一个长度为N的RDD
flatMap会将一个长度为N的RDD转换成一个N个元素的集合，然后再把这N个元素合成到一个单个RDD的结果集

union 联合

union 返回两个 RDD 并集，不去重
distinct 数据去重
intersection 返回两个RDD的交集，并且去重
subtract 返回只在 左RDD 且不在 右RDD 的数据

filter 过滤

filter 数据过滤 , 将符合条件的数据留下来

reduce 操作

reduce 将 RDD 中元素前两个传给输入函数，产生一个新的return值，新产生的return值与RDD中下一个元素（第三个元素）组成两个元素，再被传给输入函数，直到最后只有一个值为止。

sample 抽样

sample(withReplacement : scala.Boolean, fraction : scala.Double，seed scala.Long)
sample算子时用来抽样用的，其有3个参数
withReplacement：表示抽出样本后是否在放回去，true表示会放回去，这也就意味着抽出的样本可能有重复
fraction ：抽出多少，这是一个double类型的参数,0-1之间，eg:0.3表示抽出30%
seed：表示一个种子，根据这个seed随机抽取，一般情况下只用前两个参数就可以，那么这个参数是干嘛的呢，这个参数一般用于调试，有时候不知道是程序出问题还是数据出了问题，就可以将这个参数设置为定值。

takeSample

takeSample 返回一个Array, 该方法仅在预期结果数组很小的情况下使用，因为所有数据都被加载到driver的内存中

Pair RDD

Spark 为包含键值对对类型的 RDD 提供了一些专有操作，这些操作就被称为 Pair RDD，Pair RDD 是很多程序的构成要素，因为它们提供了并行操作对各个键或跨节点重新进行数据分组的操作接口。

提取每个元素的第一个元素作为 key 剩余元素作为 value 创建 Pair RDD，Pair RDD 由于有 key 的存在，与普通的 RDD 相比更加格式化，这种特性就会给 Pair RDD 赋予一些特殊的操作，例如 groupByKey() 可以将具有相同 key 进行分组，其结果仍然得到 Pair RDD，然后利用 mapValues() 对相同 key 的 value 进行函数计算，reduceByKey()、countByKey()和sortByKey()等一系列ByKey()操作

sortByKey 排序

sortByKey([ascending], [numTasks]) 升序还是降序取决于 ascending 参数

reduceByKey

reduceByKey 合并具有相同键值的值

mapValues

mapValues 对键值对RDD中的每个value都应用一个函数

keys values

keys 返回所有的键值
values 返回所有的值

join 连接

join 根据两个数据集中都有的值来连接两个 RDD， leftOuterJoin：返回左侧数据集的记录，rightOuterJoin 返回右侧数据集的记录。

reduceByKey

reduceByKey 方法和 reduce 类似，但是是在 键-值 基础上运行的

Statistics 基础统计

MLlib Statistics是基础统计模块，能对RDD数据进行统计，包括最大值(max)，最小值(min)，均值mean、方差(variance)、总数(count), L1范数normL1, L2范数normL2。
