变量作用域

发现出错了，看一下函数生成字节码时发生了什么：

如上，print(b) 语句没有执行通过，这是因为Python 在编译函数的定义体时，它先判断b是局部变量，因为在函数中给它赋值了，在生成字节码时，（10 LOAD_FAST ）加载本地名称b，表明编译器把b视作局部变量，即使在后面才为b赋值，因为变量的种类（是不是局部变量）不能改变函数的定义体。所以 Python 会尝试从本地环境中获取b，但此时b没有被赋值，在没有被赋值之前就想使用b，便会出现上面的错误。

这不是缺陷，而是设计选择：Python 不要求声明变量，但是假定在函数定义体中赋值的变量是局部变量。如果在函数中赋值时想让解释器把b当作全局变量，要用 global 声明。
