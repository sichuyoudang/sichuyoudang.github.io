Posted on 2019-09-20

Python 唯一支持的参数传递模式是共享传参。共享传参指函数的各个形式参数获得实参中各个引用的副本。也就是说，函数内部的形参是实参的别名。

bus3 一开始也是空的，因此还是默认赋值的列表，但是默认赋值的列表不是空的，此时 bus2.passengers 和 bus3.passengers 指代同一个列表，bus1.passengers 是不同的列表。

结果显示没有指定初始值的 Bus 实例会共享列表，出现这个问题的根源是，默认值在定义函数时计算（通常在加载模块时），因此默认值变成了函数对象的属性。所以如果默认值时可变对象的话，修改了它的值，那么后续的函数调用都会收到影响。

防御可变参数

如果定义的函数接受可变参数，应该谨慎考虑调用方是否期望修改传入的参数。

例如，如果函数接受一个字典，而且在处理的过程中要修改它，那么这个这个副作用要不要体现到函数外部？这需要函数的编写者和调用方法达成共识。

如上，参数bask_team也被修改了，因为语句self.passengers = passengers 是把self.passengers 变成了 passengers的别名，而 passengers 是 init 方法的实参的别名。

修正的方法是 Bus 维护自己的 passengers 列表，在init中，传人passengers参数时，应该把参数值的副本赋值给 self.passengers。

创建passengers列表的副本，如果不是列表，就把它转传成列表。
在内部这样处理，就不会影响初始化Bus时传入的参数了。此外这种方式还更灵活，现在，传给 passengers参数的值可以是元祖或者任何其他可迭代对象，如 set 对象，甚至数据库查询结果，因为list构造方法接受任何可迭代对象。

除非这个方式确实是想修改通过参数传入方对象，否则在类中直接把参数赋值给实例变量之前一定要三思，因为这样会外参数对象创建别名。如果不确定，那就创建副本。
